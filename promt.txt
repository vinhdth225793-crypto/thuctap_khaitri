# PROMPT #2 — Xóa trường `password` thừa khi tạo tài khoản

## Vấn đề
Model `NguoiDung` dùng trường `mat_khau` để lưu mật khẩu (đã khai báo `getAuthPassword()` trả về `$this->mat_khau`). Trường `password` **không có trong `$fillable`** nên Laravel tự bỏ qua khi gọi `create()`, nhưng code vẫn đang set nó — gây nhầm lẫn và tiềm ẩn lỗi nếu schema thay đổi.

**Bằng chứng — `AuthController.php` method `xuLyDangKy()`:**
```php
$nguoiDung = NguoiDung::create([
    'mat_khau' => Hash::make($request->mat_khau),
    'password' => Hash::make($request->mat_khau), // ❌ THỪA — không có trong $fillable
    ...
]);
```

**Bằng chứng — `AdminController.php` method `approveTaiKhoan()`:**
```php
$nguoiDung = NguoiDung::create([
    'mat_khau' => $taiKhoan->mat_khau,
    'password' => $taiKhoan->mat_khau, // ❌ THỪA
    ...
]);
```

---

## PHASE 2.1 — Sửa trong `AuthController.php`

**File:** `app/Http/Controllers/AuthController.php`
**Method:** `xuLyDangKy()`

Xóa dòng `'password' => Hash::make($request->mat_khau),` trong block tạo học viên.

Kết quả sau khi sửa:
```php
$nguoiDung = NguoiDung::create([
    'ho_ten'       => $request->ho_ten,
    'email'        => $request->email,
    'mat_khau'     => Hash::make($request->mat_khau), // ✅ Chỉ giữ lại trường này
    'vai_tro'      => $request->vai_tro,
    'so_dien_thoai'=> $request->so_dien_thoai,
    'ngay_sinh'    => $request->ngay_sinh,
    'dia_chi'      => $request->dia_chi,
    'trang_thai'   => true,
]);
```

# PROMPT #3 — Tạo profile GiangVien/HocVien khi Admin phê duyệt tài khoản

## Vấn đề
Khi admin phê duyệt tài khoản giảng viên trong `approveTaiKhoan()`, code chỉ tạo bản ghi `NguoiDung` nhưng **không tạo bản ghi tương ứng trong bảng `giang_vien`**.

Hậu quả: Giảng viên đăng nhập xong → vào trang profile → `$user->load('giangVien')` trả về `null` → View bị lỗi hoặc hiển thị sai.

**Code hiện tại (bị thiếu):**
```php
public function approveTaiKhoan($id)
{
    $taiKhoan = TaiKhoanChoPheDuyet::findOrFail($id);

    $nguoiDung = NguoiDung::create([...]);

    // ❌ THIẾU: không tạo bản ghi giang_vien sau đây

    $taiKhoan->update(['trang_thai' => 'da_phe_duyet']);
    ...
}
```

## PHASE 3.2 — Thêm logic tạo profile sau khi approve

**File:** `app/Http/Controllers/Admin/AdminController.php`
**Method:** `approveTaiKhoan()`

Thêm đoạn code tạo profile ngay sau khi tạo `$nguoiDung`, **bên trong DB::transaction** để đảm bảo toàn vẹn dữ liệu.

Kết quả sau khi sửa — toàn bộ method `approveTaiKhoan()`:
```php
public function approveTaiKhoan($id)
{
    $taiKhoan = TaiKhoanChoPheDuyet::findOrFail($id);

    // Dùng transaction để đảm bảo nếu 1 bước lỗi thì rollback toàn bộ
    DB::transaction(function () use ($taiKhoan) {

        // Bước 1: Tạo tài khoản người dùng
        $nguoiDung = NguoiDung::create([
            'ho_ten'        => $taiKhoan->ho_ten,
            'email'         => $taiKhoan->email,
            'mat_khau'      => $taiKhoan->mat_khau,
            'vai_tro'       => $taiKhoan->vai_tro,
            'so_dien_thoai' => $taiKhoan->so_dien_thoai,
            'ngay_sinh'     => $taiKhoan->ngay_sinh,
            'dia_chi'       => $taiKhoan->dia_chi,
            'trang_thai'    => true,
        ]);

        // Bước 2: Tạo profile tương ứng theo vai trò
        if ($taiKhoan->vai_tro === 'giang_vien') {
            GiangVien::create([
                'nguoi_dung_id' => $nguoiDung->ma_nguoi_dung,
            ]);
        }
        // Lưu ý: hoc_vien thường tự đăng ký nên không cần tạo ở đây
        // Nhưng nếu admin approve một hoc_vien thì cũng xử lý:
        // elseif ($taiKhoan->vai_tro === 'hoc_vien') {
        //     HocVien::create(['nguoi_dung_id' => $nguoiDung->ma_nguoi_dung]);
        // }

        // Bước 3: Cập nhật trạng thái phê duyệt
        $taiKhoan->update(['trang_thai' => 'da_phe_duyet']);
    });

    $redirectUrl = $taiKhoan->vai_tro === 'giang_vien'
        ? route('admin.giang-vien.index')
        : route('admin.hoc-vien.index');

    return response()->json([
        'success' => true,
        'message' => 'Đã phê duyệt tài khoản ' . $taiKhoan->ho_ten . '.',
        'redirect' => $redirectUrl,
        'vai_tro'  => $taiKhoan->vai_tro
    ]);
}
```

> ℹ️ Lưu ý: `DB` facade đã được import sẵn ở đầu file (`use Illuminate\Support\Facades\DB;`) — kiểm tra lại trước khi thêm.

✅ Pass: tìm thấy dòng `GiangVien::create` trong method `approveTaiKhoan`

# PROMPT #4 — Tạo profile khi Admin tạo user thủ công

## Vấn đề
Khi admin tạo user thủ công qua form (method `storeNguoiDung()`), code chỉ gọi `NguoiDung::create($data)` nhưng **không tạo bản ghi profile** tương ứng trong `giang_vien` hoặc `hoc_vien`.

Hậu quả: User được tạo nhưng không có profile → vào trang profile bị lỗi.

**Code hiện tại (bị thiếu):**
```php
public function storeNguoiDung(Request $request)
{
    ...
    NguoiDung::create($data); // ❌ Tạo xong không tạo profile

    return redirect()->route('admin.tai-khoan.index')
        ->with('success', 'Tạo người dùng mới thành công.');
}
```

---

## PHASE 4.1 — Sửa method `storeNguoiDung()` trong AdminController

**File:** `app/Http/Controllers/Admin/AdminController.php`
**Method:** `storeNguoiDung()`

Thay thế đoạn `NguoiDung::create($data)` thành:

```php
DB::transaction(function () use ($data, $request) {
    $nguoiDung = NguoiDung::create($data);

    // Tạo profile tương ứng theo vai trò
    if ($request->vai_tro === 'giang_vien') {
        GiangVien::create([
            'nguoi_dung_id' => $nguoiDung->ma_nguoi_dung,
        ]);
    } elseif ($request->vai_tro === 'hoc_vien') {
        \App\Models\HocVien::create([
            'nguoi_dung_id' => $nguoiDung->ma_nguoi_dung,
        ]);
    }
    // admin không cần profile riêng
});

return redirect()->route('admin.tai-khoan.index')
    ->with('success', 'Tạo người dùng mới thành công.');
```

> ℹ️ Biến `$request` cần được truyền vào closure qua `use ($data, $request)` để dùng `$request->vai_tro` bên trong.

# PROMPT #5 — Thêm Rate Limiting chống brute-force cho trang đăng nhập

## Vấn đề
Route POST `/dang-nhap` hiện không có giới hạn số lần thử. Kẻ tấn công có thể thử mật khẩu liên tục (brute-force) mà không bị chặn.

**Code hiện tại trong `routes/web.php`:**
```php
Route::middleware('guest')->group(function () {
    Route::get('/dang-nhap', [AuthController::class, 'showDangNhap'])->name('dang-nhap');
    Route::post('/dang-nhap', [AuthController::class, 'xuLyDangNhap'])->name('xu-ly-dang-nhap'); // ❌ Không có throttle
    ...
});
```

---

## PHASE 5.1 — Thêm throttle vào route đăng nhập trong `routes/web.php`

**File:** `routes/web.php`

Sửa route POST `/dang-nhap` thành:
```php
// Giới hạn: tối đa 5 lần thử / 1 phút / mỗi IP
Route::post('/dang-nhap', [AuthController::class, 'xuLyDangNhap'])
    ->name('xu-ly-dang-nhap')
    ->middleware('throttle:5,1');
```

> ℹ️ `throttle:5,1` = 5 lần / 1 phút. Laravel tự trả về HTTP 429 khi vượt ngưỡng.

### Kiểm tra hoàn thành Phase 5.1
```bash
grep -n "throttle" routes/web.php
```
✅ Pass: tìm thấy dòng `throttle` gắn vào route `xu-ly-dang-nhap`

---

## PHASE 5.2 — Hiển thị thông báo thân thiện khi bị chặn

**File:** `resources/views/pages/auth/dang-nhap.blade.php`

Thêm đoạn hiển thị lỗi 429 ngay trong `<x-alert />` hoặc trước form:

```blade
@if (session()->has('errors') || $errors->has('email'))
    {{-- đã có x-alert xử lý --}}
@endif

{{-- Thông báo khi bị throttle (HTTP 429) --}}
@error('throttle')
    <div class="alert alert-danger" role="alert">
        <i class="fas fa-exclamation-triangle me-2"></i>
        Bạn đã nhập sai quá nhiều lần. Vui lòng thử lại sau <strong>1 phút</strong>.
    </div>
@enderror
```

> ℹ️ Laravel tự inject lỗi throttle vào session errors với key `throttle` khi dùng middleware `throttle`. Nếu component `x-alert` đã hiển thị tất cả lỗi thì bước này có thể bỏ qua.

### Kiểm tra hoàn thành Phase 5.2
Thử đăng nhập sai 6 lần liên tiếp.
✅ Pass: lần thứ 6 bị chặn, server trả về HTTP 429 hoặc hiển thị thông báo lỗi
❌ Fail: vẫn được thử không giới hạn
